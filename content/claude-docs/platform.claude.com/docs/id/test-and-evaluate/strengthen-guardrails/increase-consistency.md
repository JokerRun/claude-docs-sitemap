# Meningkatkan konsistensi output

Buat output Claude lebih konsisten dengan menentukan format yang tepat, mengisi awal respons, membatasi dengan contoh, dan mendasarkan jawaban pada retrieval.

---

<Tip>
  **Untuk kesesuaian skema JSON yang terjamin**

  Jika Anda membutuhkan Claude untuk selalu menghasilkan JSON valid yang sesuai dengan skema tertentu, gunakan [Structured Outputs](/docs/id/build-with-claude/structured-outputs) alih-alih teknik rekayasa prompt di bawah ini. Structured outputs memberikan kepatuhan skema yang terjamin dan dirancang khusus untuk kasus penggunaan ini.

  Teknik di bawah ini berguna untuk konsistensi output secara umum atau ketika Anda membutuhkan fleksibilitas di luar skema JSON yang ketat.
</Tip>

Berikut cara membuat respons Claude lebih konsisten:

## Tentukan format output yang diinginkan

Definisikan format output yang Anda inginkan secara tepat menggunakan JSON, XML, atau templat kustom sehingga Claude mengikuti setiap elemen pemformatan output yang Anda butuhkan.

<Accordion title="Contoh: Menstandarkan umpan balik pelanggan">
  | Peran     | Konten                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                        |
  | --------- | ----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | User      | Anda adalah AI Customer Insights. Analisis umpan balik ini dan hasilkan output dalam format JSON dengan kunci: "sentiment" (positive/negative/neutral), "key\_issues" (list), dan "action\_items" (list of dicts dengan "team" dan "task"). "Saya telah menjadi pengguna setia selama 3 tahun, tetapi pembaruan UI terbaru adalah bencana. Menemukan fitur dasar sekarang seperti perburuan harta karun. Selain itu, harga 'premium' yang baru sangat keterlaluan. Saya mempertimbangkan untuk beralih kecuali ini diperbaiki secepatnya."                                    |
  | Assistant | \{ "sentiment": "negative", "key\_issues": \[ "Poor UI/UX in recent update", "Difficulty finding basic features", "Perceived high pricing (new 'premium' tier)" ], "action\_items": \[ \{ "team": "Product", "task": "Conduct usability testing and iterate on UI based on findings" }, \{ "team": "UX", "task": "Create a feature discovery guide or onboarding flow" }, \{ "team": "Marketing", "task": "Review pricing strategy and communicate value proposition" }, \{ "team": "Customer Success", "task": "Reach out to user, offer concessions to prevent churn" } ] } |
</Accordion>

## Gunakan retrieval untuk konsistensi kontekstual

Untuk tugas yang membutuhkan konteks yang konsisten (misalnya, chatbot, basis pengetahuan), gunakan "retrieval" (pengambilan) untuk mendasarkan respons Claude pada kumpulan informasi yang tetap.

## Rangkai prompt untuk tugas kompleks

Pecah tugas kompleks menjadi subtugas yang lebih kecil dan konsisten. Setiap subtugas mendapatkan perhatian penuh Claude, mengurangi kesalahan inkonsistensi di seluruh alur kerja berskala besar.

## Jaga Claude tetap dalam karakter

Untuk aplikasi berbasis peran, mempertahankan karakter yang konsisten membutuhkan prompting yang disengaja.

* **Gunakan prompt sistem untuk menetapkan peran:** Gunakan [prompt sistem](/docs/id/build-with-claude/prompt-engineering/claude-prompting-best-practices#give-claude-a-role) untuk mendefinisikan peran dan kepribadian Claude. Ini menetapkan fondasi yang kuat untuk respons yang konsisten.
  <Tip>
    Saat menyiapkan karakter, berikan informasi terperinci tentang kepribadian, latar belakang, dan sifat atau keunikan tertentu. Ini membantu model meniru dan menggeneralisasi sifat karakter dengan lebih baik.
  </Tip>
* **Persiapkan Claude untuk skenario yang mungkin terjadi:** Berikan daftar skenario umum dan respons yang diharapkan dalam prompt Anda. Ini "melatih" Claude untuk menangani situasi yang beragam tanpa keluar dari karakter.

<Accordion title="Contoh: Chatbot perusahaan untuk prompting peran">
  | Peran  | Konten                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                                               |
  | ------ | -------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
  | System | Anda adalah AcmeBot, asisten AI kelas perusahaan untuk AcmeTechCo. Peran Anda: - Menganalisis dokumen teknis (TDD, PRD, RFC) - Memberikan wawasan yang dapat ditindaklanjuti untuk tim engineering, produk, dan operasional - Mempertahankan nada yang profesional dan ringkas                                                                                                                                                                                                                                                                                                                                                                                       |
  | User   | Berikut kueri pengguna untuk Anda respons: \<user\_query> \{\{USER\_QUERY}} \</user\_query> Aturan interaksi Anda adalah: - Selalu merujuk pada standar AcmeTechCo atau praktik terbaik industri - Jika tidak yakin, minta klarifikasi sebelum melanjutkan - Jangan pernah mengungkapkan informasi rahasia AcmeTechCo. Sebagai AcmeBot, Anda harus menangani situasi sesuai pedoman berikut: - Jika ditanya tentang IP AcmeTechCo: "I cannot disclose TechCo's proprietary information." - Jika ditanya tentang praktik terbaik: "Per ISO/IEC 25010, we prioritize..." - Jika tidak jelas tentang suatu dokumen: "To ensure accuracy, please clarify section 3.2..." |
</Accordion>
